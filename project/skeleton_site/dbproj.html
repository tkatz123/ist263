Restaurant SQL Database

This project demonstrates the full creation of a normalized SQL database from CSV files. Using SQLite, I built a restaurant analytics database from the ground up — from ERD design to table creation, data population via Python, and advanced SQL queries analyzing customer behavior, revenue trends, and restaurant performance.

Dataset

Sourced from Kaggle
The dataset includes 10 CSV files detailing members, restaurants, meals, orders, payments, etc.

Methodology

  1. Schema Design
    • Created a conceptual ERD in draw.io
    • Built a physical ERD in Vertabelo, defining keys, types, constraints, and relationships
    • Resolved many-to-many relationships and set up foreign key constraints
  2. Database Construction
    • Generated SQL scripts to create and drop tables in dependency order
    • Used Python to convert CSVs into INSERT statements (workaround for DB Browser limitations)
  3. Query Development
    • Wrote themed SQL scripts to explore and analyze key aspects of the restaurant business:
      • customer_behavior.sql – member activity and spending
      • meal_popularity.sql – top-selling items and category insights
      • restaurant_performance.sql – metrics per restaurant
      • monthly_trends.sql – seasonal business trends
      • revenue_analysis.sql – revenue drivers and order value
      • views_and_ctes.sql – modular views and CTE examples
      • data_validation.sql – full-table joins for schema integrity checks

Results

  • Successfully created a fully normalized SQL database from scratch
  • Joined and validated data from 10 tables using SQL
  • Produced modular, reusable SQL scripts to answer business questions across multiple domains
  • Used CTEs and views to streamline complex queries and improve clarity
  • Demonstrated expertise in SQL data modeling, ingestion, and querying in a local environment without a hosted database
